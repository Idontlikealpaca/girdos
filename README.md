# GridOS

마이크로그리드 에너지 디스패치 최적화 플랫폼

## 개요

GridOS는 태양광 발전과 배터리 저장 시스템을 갖춘 마이크로그리드에서 에너지 디스패치를 최적화하는 웹 기반 플랫폼입니다. NSGA-II (Non-dominated Sorting Genetic Algorithm II)를 사용하여 비용, 탄소 배출, 배터리 열화를 동시에 최적화합니다.

## 기능

- **다목적 최적화**: 비용 최소화, 탄소 배출 최소화, 배터리 열화 최소화
- **실시간 시각화**: 파레토 프론트, 디스패치 스케줄, 에너지 밸런스
- **몬테카를로 시나리오**: 태양광 발전 변동성 고려
- **Web Worker 기반**: UI 블로킹 없는 백그라운드 최적화

## 알고리즘 설명

### NSGA-II (Non-dominated Sorting Genetic Algorithm II)

1. **염색체 표현**: 24시간 배터리 충방전 명령 (-1 ~ 1, 최대 ±30kW)
2. **목적 함수**:
   - 비용: KEPCO TOU 요금제 기반 전력 구매 비용
   - 탄소: 전력 구매량 × 탄소 계수 (0.4596 kgCO₂/kWh)
   - 열화: 배터리 에너지 처리량 (절대값 합)
3. **제약 조건**: SoC 10%-95%, 배터리 최대 출력 30kW
4. **진화 연산자**:
   - 선택: 비지배 정렬 + 혼잡도 거리 기반 토너먼트
   - 교차: 2점 교차
   - 돌연변이: 가우시안 (rate=0.12, σ=0.3)
5. **파라미터**: 인구수 80, 세대 40

### 몬테카를로 시나리오 생성

태양광 기저 발전량에 0.6~1.4배 변동 계수를 균등 분포로 적용하여 80개 시나리오 생성.

## 시스템 파라미터

- **수요 프로파일**: [18,16,15,14,14,15,20,28,38,48,52,55,53,52,50,47,44,40,42,44,38,30,24,20] kW
- **태양광 기저**: [0,0,0,0,0,0,2,8,18,30,38,42,44,42,38,30,18,8,2,0,0,0,0,0] kW
- **배터리**: 용량 100kWh, 최대 출력 30kW, SoC 제약 10%-95%
- **KEPCO TOU**:
  - 피크 (9-12, 13-18시): ₩160/kWh
  - 중간 (7-9, 18-23시): ₩112/kWh
  - 경부하: ₩67/kWh
- **탄소 계수**: 0.4596 kgCO₂/kWh

## 실행 방법

```bash
npm install
npm run dev
```

브라우저에서 http://localhost:5173 접속.

"Run Optimization" 버튼을 클릭하여 최적화 실행. 완료 후 파레토 차트에서 해를 선택하여 세부 정보 확인.

## 배포

Vercel을 위한 vercel.json 포함.

```bash
npm run build
```

빌드된 dist/ 폴더를 Vercel에 업로드.

## 기술 스택

- React 18 + Vite + TypeScript
- Recharts (차트 라이브러리)
- Zustand (상태 관리)
- CSS Modules (스타일링)
- Web Workers (백그라운드 계산)
